# Project Setup Workflow

ใช้เมื่อเริ่มงานใดๆ ก็ตาม

## ลำดับการทำงาน

### 1. รับคำสั่งและสร้าง Issue
เมื่อได้รับคำสั่ง ก่อนลงมือทำให้สร้าง issue ใน github แล้วให้ลงมือทำตามคำสั่งทันที

### 2. ตรวจสอบและปรับโค้ดเบส
ตรวจสอบโค้ดเบสแล้วปรับใช้ให้เหมาะสมกับคำสั่งอย่างเคร่งครัด

### 3. เสนอข้อแนะนำ
เมื่อมีข้อโต้แย้งหรือคำแนะนำใดๆ ที่เป็นประโยชน์ในทางบวก หรือมีประสิทธิภาพ ให้นำเสนอ

### 4. สร้าง UI ก่อน
เมื่อสร้างฟีเจอร์ใดๆ ให้เริ่มสร้างจาก UI ก่อนเสมอ และเมื่อสร้าง UI ให้ทำตามกฎที่เกี่ยวข้องกับ UI เช่น mobile-first หรือ shadcn

### 5. สร้างสิ่งอื่นๆ ตามลำดับ
เมื่อสร้าง UI แล้วให้สร้างสิ่งอื่นๆ ตามลำดับสิ่งที่ต้องทำเพื่อให้การเพิ่มฟีเจอร์ในครั้งนี้ประสบผลสำเร็จ

### 6. ทดสอบก่อนรายงาน
เมื่อทำฟีเจอร์ที่ได้รับมอบหมายเสร็จสิ้นทุกครั้ง ก่อนจะรายงานผลให้ทำเทสก่อนเสมอ
- เทส UI ด้วย playwright
- เทสอื่นๆ ด้วย curl หรือ vitest

### 7. อัพเดตเอกสารและเวอร์ชัน
สร้างเอกสารของส่วนที่เพิ่มเติมเข้ามา และอัพเดตเวอร์ชันของโปรเจ็ค
- ถ้าเป็นฟีเจอร์เล็กน้อย: +v0.0.1
- ถ้าเป็นฟีเจอร์ใหญ่: +v0.1.0

### 9. รายงานผล
รายงานผลให้ผู้ใช้ทราบเป็นภาษาไทยให้เข้าใจง่าย
